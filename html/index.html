關於

沃羅諾伊圖(Voronoi Diagram)為一個演算法可以幫助我們空間分割，可應用於幾何、建築地理學 等。其特色為圖上任意點落於哪個區域(黑色線段所圍區域)中則與該點距離最近的紅點必也在 該區域中。

軟體規格書

輸出與輸入(資料)規格

輸入資料要求為須輸入測試點數，再逐一輸入點的x座標與y座標，輸入點數為0時表示測試結束。 輸出會依照不同的圖片輸出不同的檔案，在每行前面加上P或E表示為point(點)或edge(邊)。

功能規格與介面規格

主介面如下圖
read file : 讀取原始檔案
read file(old) : 讀取輸出檔案
next set : 顯示下個點集合
write file : 將目前的點與線輸出
play : 將voronoi diagram一次性完成
step by step : 依步驟完成voronoi diagram
clear : 將全畫面清空

軟體測試規劃書

本程式為使用python設計，並用python套件tkinter做為視窗介面工具。 其中voronoi diagram的以遞迴方式解，而遞迴最小單位為2點。 對於任何大於2點的點集合會將點結合分隔成兩半直至點數為2為止。 再將較小點集合的voronoi diagram做合併直至原點和為止。

程式設計

資料結構

node存放點的資訊：
。x : 點的x座標
。y : 點的y座標
。line_index : 將該點圍住的線集合
。node_id : tkinter上點的編號

line存放線段的資訊 :
。x : 線段的起點x座標
。y : 線段的起點y座標
。ex : 線段的終點x座標
。ey : 線段的終點y座標
。vx : 線段的x向量
。vy : 線段的y向量
。a, b ,c : 線段方程式的係數(ax+by+c=0)
。node_index : 被線段分開的兩點
。line_id : tkinter上線段的編號

程式設計

將要解的問題分成兩類，點數小於等於2與點數大於2。
若點數小於等於2直接畫一條中垂線即可(1個點無中垂線)。
若點數大於2則需先將點集合拆半分開做兩次voronoi diagram再做合併， 若拆半後點數仍大於2就繼續拆半直至點數小於等於2。

2點

4點

1. 將4點拆成2組2點並各自完成voronoi diagram。

2. 將2組2點以convex hull合併並找出上下切線，以方便下階段產生hyper-plan。

2. 將2組2點以hyper-plan分隔，即完成4點voronoi diagram。

消線

如圖當點集合兩邊皆為鈍角三角形時最多共有6條線段需要被消除，然而在一般情況下至多只需消除4條線段 ，因此在每次合併時需要做額外的檢查動作，避免線段沒有消除乾淨殘留在畫布上影響下次合併。

心得

voronoi diagram使用DP方式蠻考驗同學的資料結構能力與演算法設計能力， 在我實做該演算法時，資料結構前前後後就被我更改了許多次為了能更好的完成問題。 就算如此我的演算法仍有改進空間，有許多問題尚未被解決完全，像是碰上垂直與水平線時 因我的演算法會參考線段的斜率而使我的結果不如預期；又或著當點集合數量增加時 ，就會有新的消線規則須思考，我原先消線的方法就會出現漏洞。
